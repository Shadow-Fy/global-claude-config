# 深度模块

来自《软件设计的哲学》：

**深度模块** = 小接口 + 大量实现

```
┌─────────────────────┐
│   小接口 (Interface) │  ← 少方法，简单参数
├─────────────────────┤
│                     │
│                     │
│  深度实现 (Implementation)│  ← 隐藏的复杂逻辑
│                     │
│                     │
└─────────────────────┘
```

**浅层模块** = 大接口 + 少实现（避免）

```
┌─────────────────────────────────┐
│       大接口 (Interface)        │  ← 多方法，复杂参数
├─────────────────────────────────┤
│  薄实现 (Implementation)        │  ← 只是透传
└─────────────────────────────────┘
```

设计接口时，问：

- 我能减少方法的数量吗？
- 我能简化参数吗？
- 我能在内部隐藏更多复杂性吗？
