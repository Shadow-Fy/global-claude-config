# 完善 Issue 描述

针对目标 issue，我们需要先研究目标 issue ，了解上下文后，整理相关信息必要时完善 issue 内容，方便后续继续讨论/制作。

合理设置 label、type

合理设置标题，一个好的标题应当简洁明了地反映任务内容，方便后续检索和归档。例如：`SU：怪物：新小怪：书灵`

## issue 内容格式

必须包含以下部分（需要保持顺序）：

1. 任务列表。每个子任务一行 `- [] 子任务 A`
2. 现状
3. 期望
4. 技术方案

## 输出

1. issue 链接
2. 研究报告
3. 完善后的 issue 内容，询问用户是否需要将更新后的内容同步到 issue
