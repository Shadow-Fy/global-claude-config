# 提交暂存的更改

提交目标仓库（上文修改的仓库，如果没有上文，则为当前仓库）暂存的更改。
不要修改代码或者回滚任何代码，需要提交的代码已经被 `git add` 添加到暂存区。

## 提交信息规范

- 语言：中文
- 格式：`<type>: <简要描述>`，type 为 feat/fix/refactor/docs/chore
- 正文（可选）：用 `-` 列出关键变更点
- 如果修复了某个 issue，末尾加 `Closes #<number>`
- 禁止携带 Co-Authored-By 等署名信息

## 执行步骤

1. 分析目标仓库是哪个。如果下文指定的仓库名，则视为目标仓库；否则视为上文中读写的仓库。
2. 执行 `pwd` 输出目标仓库的绝对路径，并确认该路径正确无误
3. cd 到目标仓库目录，方便后续执行命令
4. `git diff --cached --stat` 查看暂存内容
5. 根据上文对话上下文和 diff 内容编写提交信息
6. 提交

## 输出

仅输出`提交成功。`
